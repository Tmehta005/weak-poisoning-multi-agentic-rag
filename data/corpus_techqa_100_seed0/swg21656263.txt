Technote (troubleshooting)
Problem(Abstract)
Unable to view images in reports in PDF/Excel output when using Report Studio/Workspace.
However images are visible in HTML format when reports are run actively.
Symptom
Unable to view images in reports in PDF/Excel output when using Report Studio/Workspace.
However images are visible in HTML format when reports are run actively.
The PDF/Excel outputs will have a placeholder image. This will be in the form of an empty black box (PDF) or will be a missing pictures icon (Excel).
Cause
There are a few common causes for this issue:
The permissions of the folder which contains the image, have not been configured correctly
The image is not within the directory which is specified
The directory where the image is meant to be stored, does not exist in the specified location
Diagnosing the problem
To find out the root cause of this problem, you will need to use tools that monitor and record all actions attempted against the Operating System Registry.
In
THIS
scenario the tool used was Process Monitor and the Web Server was IIS 7.5.
Download Process Monitor tool (Make sure you run it from the same machine Cognos is installed on)
Open up the report in Cognos Workspace
Load up Process Monitor
You will notice the current processes immediately being captured and displaying in the window. Press the Capture / Ctrl+E Icon (Magnifying Glass) to stop the capture of events.
Press the Clear / Ctrl+X icon (Eraser) to remove all the recorded entries.
Press the capture icon in process monitor
Export the report to Excel 2007/PDF
Save the file anywhere
Press the Capture icon again in process monitor to stop the capturing of events.
Press the Find Icon / Ctrl+F and type "[Image Name]" to make sure the filter image event was captured.
You should be able to see the reason why the image could not be saved under the 'Result' column.
Resolving the problem
If the image is found but cannot be accessed due to permission configuration issue:
Open up IIS 7.5
Click on the root folder of your Cognos installation (E.g. C1021GA) in the navigation pane on the left side
When the root folder is selected, double-click 'Authentication'
Ensure that anonymous access is enabled
Repeat steps 3 and 4 for the image folder and it's parent folder.
If the user is concerned about security, they may want to restrict the child-folders (E.g. cgi-bin) and change the authentication settings accordingly
Run the report in export as Excel 2007 and PDF
If the image/ directory is not located:
Place the image/directory in the specified location
